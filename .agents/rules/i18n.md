# Language & Internationalization (i18n)

## Language & Internationalization (i18n) Rules

* The website language is **Persian**. All textual content, metadata, Schema.org attributes, and JSON-LD must be in Persian to optimize local SEO unless explicitly requested otherwise.
* Always refer to the [Project Brief](/docs/project-brief.md) for content placement context.
* **Middleware & Routing Integrity:** All language negotiation, initial routing, and cookie management MUST be handled at the Edge via Next.js Middleware. Components should never guess the locale; they must receive it strictly via route parameters (e.g., `[locale]`) or headers passed down from the middleware.
* **Bidirectional (BiDi) UI Strictness:** The root layout must dynamically set the `lang` and `dir` attributes on the `<html>` tag based on the active locale (e.g., `<html lang="fa" dir="rtl">`).
* **Data Formatting (Dates, Numbers, Currency):** Do not manually format numbers or dates. ALWAYS use the native JavaScript `Intl` API (`Intl.NumberFormat`, `Intl.DateTimeFormat`) for rendering currencies, decimals, and dates. For Persian locales, ensure the calendar system defaults to Jalali/Shamsi and numbers render in Persian numerals where stylistically required.
